Step-by-Step: Virtual ECU + Cloud Hardware

Run HPC / GPU / DSP / ARM workloads without local silicon using software backends, Docker, Kubernetes, Terraform, and free cloud tiers (Colab, GCP, AWS, Azure).

0. What you get

  • Hardware-dependent code path – optional CUDA (-DSDV_ENABLE_CUDA=ON)
  • Virtual ECU – cloud-storable image binding CPU / NVIDIA GPU / ARM / DSP / Cloud HPC
  • Free / low-cost runtimes – Google Colab T4, GCP/AWS/Azure free tiers, Docker Desktop, K8s Jobs
  • IaC – Terraform skeletons under cloud/terraform/{aws,gcp,azure}
Local PC (no GPU) └─ CPU_Sim / ARM_Sim / DSP_Sim / GPU_Emulated Docker Desktop ──► same binaries in container Google Colab (free T4) ──► build+run notebook GCP / AWS / Azure ──► Artifact Registry + K8s Job / Cloud Run / Fargate Terraform ──► provision registry (push image once)

1. Local Windows – install tools & run all demos

  1. Install (if missing):
    winget install Kitware.CMake
    winget install MartinStorsjo.LLVM-MinGW.UCRT
    winget install Docker.DockerDesktop
    winget install Hashicorp.Terraform
  2. Open a new PowerShell, then:
    cd e:\Sami\automotive_end_to_end
    powershell -ExecutionPolicy Bypass -File .\scripts\install_and_run_all.ps1
  3. Or manual:
    mkdir build; cd build
    cmake .. -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Release -DCMAKE_CXX_COMPILER=g++
    cmake --build . -j 8
    .\sdv_demo.exe
    .\sdv_usecases.exe

2. Linux / WSL / Colab shell

chmod +x scripts/install_and_run_all.sh
./scripts/install_and_run_all.sh

3. Docker (no cloud account)

  1. Start Docker Desktop.
  2. Build Virtual ECU image:
    cd e:\Sami\automotive_end_to_end
    docker build -f cloud/docker/Dockerfile -t embedailabs/sdv-virtual-ecu:latest .
  3. Run demos:
    docker run --rm embedailabs/sdv-virtual-ecu:latest sdv_demo
    docker run --rm embedailabs/sdv-virtual-ecu:latest sdv_usecases
  4. Compose:
    docker compose -f cloud/docker/docker-compose.yml up --build
    docker compose -f cloud/docker/docker-compose.yml --profile usecases up

4. Google Colab – free NVIDIA GPU

  1. Open colab.research.google.com
  2. Upload cloud/colab/sdv_virtual_ecu.ipynb
  3. Runtime → Change runtime type → T4 GPU (free)
  4. Upload/clone project to /content/sdv
  5. Run all cells → builds + sdv_demo + sdv_usecases
Colab is the fastest free path to a real NVIDIA GPU for the Virtual ECU perception kernels.

5. Kubernetes

kubectl apply -f cloud/kubernetes/sdv-jobs.yaml
kubectl -n sdv-virtual-ecu logs job/sdv-demo-job -f
kubectl -n sdv-virtual-ecu logs job/sdv-usecases-job -f

Works on Docker Desktop Kubernetes, kind/minikube, GKE Autopilot, EKS Fargate, AKS.

6. Terraform – AWS / GCP / Azure registries

AWS

cd cloud/terraform/aws
terraform init
terraform apply
# follow output next_steps to push image to ECR

GCP

cd cloud/terraform/gcp
terraform init
terraform apply -var="project_id=YOUR_GCP_PROJECT"
# or skip TF and use Colab (section 4)

Azure

cd cloud/terraform/azure
terraform init
terraform apply
# push to ACR then az container create / AKS

7. Optional real CUDA SDK

  1. Install CUDA Toolkit
  2. Rebuild:
    cmake -S . -B build -DSDV_ENABLE_CUDA=ON -DCMAKE_BUILD_TYPE=Release
    cmake --build build -j 8
  3. Without toolkit, GPU path stays emulated and still runs everywhere.

8. ARM / DSP without boards

  • ARM: software backend today; for real ISA use QEMU user-mode, AWS Graviton, or Azure Ampere VMs with the same Docker image (multi-arch build).
  • DSP: C7x/Hexagon-class MAC engine is simulated in modules/virtual_ecu; swap to vendor SDK later behind IHwBackend.
