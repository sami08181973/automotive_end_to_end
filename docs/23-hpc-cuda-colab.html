1. Module layout

modules/hpc/
  hpc.hpp / hpc.cpp          # Architecture module
  hpc_compute.hpp/.cpp     # Suite orchestration
  hpc_kernels_cpu.cpp      # Optimized CPU (tiling, unroll, fusion)
  cuda/hpc_kernels.cu      # NVIDIA CUDA kernels
  cuda/hpc_cuda_stub.cpp   # Used when CUDA off
usecases/hpc_usecases_runner.cpp  # sdv_hpc binary

2. Realtime use-cases & optimizations

Use-caseSDV relevanceOptimization
camera_preprocessVision pipelineFused scale+bias, 4-wide unroll; CUDA grid-stride

Each use-case reports naive vs optimized latency (ms), speedup×, and MOPS. When CUDA is built, a third cuda row is added.

cmake .. -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Release -DCMAKE_CXX_COMPILER=g++
cmake --build . -j 8
.\sdv_hpc.exe
.\sdv_demo.exe
.\sdv_usecases.exe

4. How to run on Google Colab (free NVIDIA T4) – step by step

  1. Open Google Colab.
  2. Runtime → Change runtime type → T4 GPU (free).
  3. Upload project zip to /content/sdv or clone your repo.
  4. Upload notebook cloud/colab/sdv_hpc_cuda.ipynb and Run all.
  5. Key cells (manual equivalent):
    !nvidia-smi
    !apt-get update -qq && apt-get install -y -qq build-essential cmake g++ nvidia-cuda-toolkit
    %cd /content/sdv
    !cmake -S . -B build -DCMAKE_BUILD_TYPE=Release -DSDV_ENABLE_CUDA=ON
    !cmake --build build -j$(nproc)
    !./build/sdv_hpc
    !./build/sdv_demo
    !./build/sdv_usecases
  6. Confirm logs show CUDA HPC kernels ENABLED and rows with variant cuda.
If nvcc is missing on Colab, install nvidia-cuda-toolkit or use the notebook cell that installs it. CPU-optimized path still runs if CUDA configure fails.

5. Enable CUDA on a local NVIDIA machine

  1. Install CUDA Toolkit so nvcc is on PATH.
  2. cmake -S . -B build -DSDV_ENABLE_CUDA=ON -DCMAKE_BUILD_TYPE=Release
    cmake --build build -j 8
    ./build/sdv_hpc

6. Latency reduction checklist

  • Prefer optimized / cuda variants over naive for production paths.
  • Batch camera frames; keep GEMM sizes tile-aligned (16/32).
  • Use CUDA for large reductions / GEMM; keep hard-realtime control on CPU/ARM.
  • Measure with sdv_hpc before/after kernel changes.
